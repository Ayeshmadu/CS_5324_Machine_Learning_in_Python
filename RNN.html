Discuss methods of tokenization in your dataset as well as any decisions to force a specific length of sequence.¶

Quora insincere Questions Classification dataset. This is a text dataset. Quora is a platform that empowers people to learn from each other. On Quora, people can ask questions and connect with others who contribute unique insights and quality answers but some people use this flatfoam to ask toxic and divisive content. So quora wants to tackele these contents and remove them automatically from the website. This is binary type of problem. 1 is taken as insincere and 0 is taken as sincere contents.

This is very large dataset taht includes 1.3 Million data points. First I tried to run whole dataset but every time after 3-4 hour my Kernek was restarted because of my laptop coudn't support this type of large RNN model. So I had to reduce the data into effortable amount. I took 20000 data points and which include exactly same 1 and 0 contents. I removed "qid" column because it's just identification. Below I mentioned the datatypes of my final dataframe.

Tokenization is a way of separating a piece of text into smaller units called tokens. Here, I have little midium size texts so I kept the max artical length as 1000 because it covers all the words in each contents. Tokenization gives the specific integer for each word and then sequence makes sentence as correct order. Finally pad sequences add the zeros for short sentences to make the all the contents as same length. I didn't use special options and use the default parameters because I just need to identify the toxic and divisive words in the content.

I have nearly same 10000 amount of images for each class. During the each round Kfold uses one fold as test and the remaining 9 as training. However, in shufflesplit, during each round n it will only use training and testing set from iteration n. When the datset bigger and bigger, cross validation time also increases, making the shuffle split a more attactive method. I choose Shuffle split.

It allows the training converge fast, prevents bias during the training and prevents the model from learning the order of the training. I have reduced my dataset size to 20000. So I can use some repeated shuffling.

EMBED_SIZE = 50
# the embed size should match the file you load glove from
embeddings_index = {}
f = open('glove.6B.50d.txt')
# save key/array pairs of the embeddings
#  the key of the dictionary is the word, the array is the embedding
for line in f:
    values = line.split()
    word = values[0]
    coefs = np.asarray(values[1:], dtype='float32')
    embeddings_index[word] = coefs
f.close()

print('Found %s word vectors.' % len(embeddings_index))

# now fill in the matrix, using the ordering from the
#  keras word tokenizer from before
found_words = 0
embedding_matrix = np.zeros((len(word_index) + 1, EMBED_SIZE))
for word, i in word_index.items():
    embedding_vector = embeddings_index.get(word)
    if embedding_vector is not None:
        # words not found in embedding index will be ALL-ZEROS
        embedding_matrix[i] = embedding_vector
        found_words = found_words+1

print("Embedding Shape:",embedding_matrix.shape, "\n",
      "Total words found:",found_words, "\n",
      "Percentage:",100*found_words/embedding_matrix.shape[0])
